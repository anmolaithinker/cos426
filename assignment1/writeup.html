Implementation Notes

  • Black and White: The luminance of each pixel is calculated and the red, green and blue components of each pixel are set to this luminance.
  • Gamma: For each pixel and its red/green/blue component, the value is taken to the power of gamma (v^gamma).
  • Saturation: A greyscale version of the image is produced. For each pixel, a linear combination of the original pixel and the greyscale pixel is output. To decrease the saturation it uses a mixture of the original and the grey pixel, and to increase the saturation it takes a larger multiple of the original and subtracts some of the grey pixel. The formula used is: output = (1-factor)*grey + factor*original, where factor>1 is increased saturation and the opposite is decreased saturation.
  • Contrast: A solid grey image is produced with the grey value being the average luminance of the original image. For each pixel, a linear combination of the original pixel and the grey pixel is output. To decrease the contrast it uses a mixture of the original and the grey pixel, and to increase the contrast it takes a larger multiple of the original and subtracts some of the grey pixel. The formula used is: output = (1-factor)*grey + factor*original, where factor>1 is increased contrast and the opposite is decreased contrast.
  • Extract Channel: For each pixel, color channels other than the one selected are set to zero.
  • Gaussian Blur: For each pixel in the output image, we take a rectangular region centered at the output pixel from the source image (size being six sigma, as by this point the contribution is minimal) to convolve with. The convolution adds these source pixels with a weight derived from a Gaussian function (exp(-dist^2/2/sigma^2)) to the target pixel. Lastly this target pixel is normalized to keep the values between 0 and 1.
  • Sharpen: A blurred version of the image is produced (using Gaussian blur with sigma=2.0). For each pixel, double the original pixel value minus the blurred pixel value is output. The formula used is: output = 2*original - blurred.
  • Edge Detect: For each pixel in the output image, we take look at 9 pixels in the source image in the shape of 3x3 square centered at the target pixel position. We take eight times the center pixel's value and subtract from this each of the surrounding pixels. Lastly this target pixel is divided by sixteen in order to normalize.
  • Motion Blur: For each pixel in the output image, we take a single pixel width line of pixels from source image, originating at the position of the target pixel and extending to the left. These pixels are added together with a weight corresponding to the distance away from the target pixel position and then the value normalized to keep the range between 0 and 1. The weights are determined by a linear ramp where the right hand side has the most weight.
  • Median: We look at a rectangular region in the source image surrounding the target pixel of size 2sigma x 2sigma. For each channel we add all the pixel values to an array, which is sorted and then the median value chosen to be used in the output image.
  • Bilateral: For each pixel in the output image, we take a rectangular region centered at the output pixel from the source image (size being six domain sigma, as by this point the contribution is minimal) to convolve with. The convolution adds these source pixels with a weight derived from the product of two Gaussian function to the target pixel. The first Gaussian function is a function of distance to the target pixel (exp(-dist^2/2/domainsigma^2)) and the second Gaussian function is a function of the visual similarity (exp(-similarity^2/2/rangesigma^2)). Similarity is defined as square root of the sum of the differences in pixel channel values squared. Lastly this target pixel is normalized to keep the values between 0 and 1.
  • Sample (nearest neighbour): This finds the closest pixel (via round) in the source image to the position requested and returns it.
  • Sample (bilinear): This looks at the four pixels immediately surrounding the requested position. At first a linear combination of the top two are created using (1-dx) as the weights, where dx is the x distance from the pixel to the requested position. Another combination is made for the bottom two. Finally these two combinations are combined using (1-dy) as the weights, where dy is the y distance from the pixel to the requested position. This final combination is returned.
  • Sample (gaussian): This looks at pixels surrounding the requested position in the source image (a rectangle of 6 sigma x 6 sigma). Each one is summed together with a weight defined by the Gaussian function (exp(-dx^2/2/sigmay^2)exp(-dy^2/2/sigmay^2)), where dx is the x distance from the pixel being summed and the requested position and dy is the y distance from the pixel being summed and the requested position. Finally this sum is normalized so that the values are between 0 and 1 and returned.
  • Scale: The new sized image is created by multiplying the width by sx and the height by sy. For each pixel in the new target image, we calculate using sx and sy the pixel position in the source image and we pass this to the sample function (described above) to create the target pixel.
  • Rotate: First the canvas is expanded and filled with black in order to be able to keep the entire rotated image within view. For each target pixel the source pixel position is calculated by x = (x' - center_x') cos(-angle) - (y' - center_y') sin(-angle) + center_x' and y = (x' - center_x') sin(-angle) + (y' - center_y') cos(-angle) + center_y'. This performs a rotation about the center. This pixel position is passed to the sample function (described above) to create the target pixel.
  • Quantize: For each pixel and each of its red/green/blue component, the value is rounded to the closest level. The number of levels is 2^n, where n is the number of bits.
  • Random Dither: For each pixel and each of its red/green/blue component, the value minus the level just below its value (floor) is compared to a random number from zero to one. If it is larger than the random number, the pixel is set to the level above its value (ceil), and if it is smaller, then it is set to the level below its value (floor). The number of levels is 2^n, where n is the number of bits.
  • Ordered Dither: For each pixel and each of its red/green/blue component, the value minus the level just below its value (floor) is compared to a value from the pattern. The pattern is a 4x4 matrix and to lookup a value we use x mod 4 and y mod 4. If it is larger than the pattern value, the pixel is set to the level above its value (ceil), and if it is smaller, then it is set to the level below its value (floor). The number of levels is 2^n, where n is the number of bits.
  • Floyd-Steinberg Dither: For each pixel and each of its red/green/blue component, the value is rounded to the closest level. An quantization error is calculated by subtracting the rounded value from the original value and is diffused to neighbouring pixels by this formula: the pixel to the right is increased by 7/16 of the error, the pixel to the bottom is increased by 5/16 of the error, the pixel to the bottom right is increased by 1/16 of the error and the pixel to bottom left is increased by 3/16 of the error. The number of levels is 2^n, where n is the number of bits.
  • Fun: This makes ripples in the image. For each target pixel the source pixel position is calculated by x = x' + 10 cos(2 pi y' / 20) and y = y'. This causes waves of period 20 vertical pixels and amplitude 10 horizontal pixels. This pixel position is passed to the sample function (described above) to create the target pixel.
  • Sobel: For each target pixel two linear combinations of the 8 surrounding pixels and the source pixel are made, according to these two matrices: [[-1, 0, +1], [-2, 0, +2], [-1, 0 +1]] and [[-1, -2, -1], [0, 0, 0], [+1, +2, +1]]. These two linear combinations are then normalized, squared, added together and square rooted. This final value is set as the R/G/B values of that target pixel.
